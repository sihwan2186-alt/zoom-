# 화상회의 취약점 분석 - 암호화

## 취약점 개요
- 일부 화상회의 서비스는 전송 구간 암호화와 종단 간 암호화(E2EE)의 보안 경계가 다를 수 있음
- 종단 간 암호화(E2EE)가 선택 기능이거나 회의 정책에 따라 비활성화될 수 있음
- 회의 키 관리가 중앙화될 경우 단일 실패점 발생 가능

## 주요 취약점
1. **E2EE 미흡**: 기본 암호화는 서버 측 복호화 가능
2. **키 관리 집중**: 중앙화된 키 관리로 단일 장애점
3. **프로토콜 취약점**: SRTP에서 취약점 발견 사례

## 보완 방안
- WebRTC E2EE 또는 DTLS-SRTP 기반 보호 적용
- 분산형 키 관리 (각 참여자가 독립 키)
- DTLS-SRTP 기반 암호화

## 구현 파일
- `encryption.py`: AES-256, RSA-4096 암호화 모듈
