# Zoom 취약점 분석 - 암호화

## 취약점 개요
- Zoom은 기본적으로 AES-256-GCM 암호화 사용
- 종단 간 암호화(E2EE)가 선택적이며, 기본적으로 비활성화
- 회의 키 관리 중앙화 → 단일 실패점 발생

## 주요 취약점
1. **E2EE 미흡**: 기본 암호화는 서버 측 복호화 가능
2. **키 관리 집중**: 중앙화된 키 관리로 단일 장애점
3. **프로토콜 취약점**: SRTP에서 취약점 발견 사례

## 보완方案 (Jitsi Meet 기반)
- Jitsi Meet은 기본적으로 E2EE 지원
- 분산형 키 관리 (각 참여자가 독립 키)
- DTLS-SRTP 기반 암호화

## 구현 파일
- `encryption.py`: AES-256, RSA-4096 암호화 모듈
- `key_management.py`: 분산형 키 관리
