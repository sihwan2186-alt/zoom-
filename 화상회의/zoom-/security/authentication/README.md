# Zoom 취약점 분석 - 인증

## 취약점 개요
- 회의 ID만으로 참여 가능 → 추측 공격 위험
- 비밀번호 보호 미흡
- 2FA 선택적 사용

## 주요 취약점
1. **약한 인증**: 회의 ID 추측 가능
2. **2FA 미적용**: 기본적으로 2단계 인증 미사용
3. **세션 하이재킹**: 토큰 탈취 가능

## 보완方案 (Jitsi Meet 기반)
- JWT 기반 인증
- 다단계 인증 (MFA) 지원
- Secure DOMAIN 설정

## 구현 파일
- `auth.py`: 다단계 인증 모듈
- `token.py`: JWT 토큰 관리
